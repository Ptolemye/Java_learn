# 2. 内存结构

## 2.1 程序计数器

- 采用寄存器结构，可以记住下一条JVM指令==(字节码)==执行地址的行号
- 每个线程都有自己==私有==的PC，各管各的
- PC不存在内存溢出问题



## 2.2 栈

**STACK**

线程运行所需要的内存空间，JVM会为每一个线程开辟一个虚拟机栈

**栈帧**：每个**方法**运行时所需要的内存，其中就包括了参数，局部变量，返回地址

**活动栈帧**：当前线程正在执行的那个方法所对应的栈帧。

![image-20260415222006259](./image/image-20260415222006259.png)

方法一调用方法二，方法二调用方法三，栈内情况如图所示

<img src="./image/image-20260416133447086.png" alt="image-20260416133447086" style="zoom:67%;" />



**Q1 垃圾回收会管理栈内存**

每一Q个方法调用结束后，栈帧自动出栈，不需要垃圾回收机制来回收

**Q2 方法内的局部变量是否线程安全**

- 安全，每个线程都有自己独立的栈空间，局部变量存储在栈空间的栈帧中，线程与线程之间互不影响。
- 如果是局部变量引用了对象（局部变量只是引用，实际的对象存储在堆中），并逃离了方法的访问，那就要考虑线程安全问题。(==逃逸==)

**栈内存溢出**

栈帧太大，太深（栈帧数量太多），都有可能导致栈内存溢出

## 2.3 堆

**HEAP**

- 通过`new`关键字创建的对象都会被放在堆中

特点

- 堆中的对象是线程共享的，所以都需要考虑线程安全问题
- 有垃圾回收机制
